# Office Viewer Desktop 0.5.6 — Feature Batch 3

Release date: 2026-07-23

Third feature batch from the three module roadmaps (`docs/plans/`): 13 new features across HTML, Markdown and the AI assistant.

## HTML

- Local HTML validation (问题 panel): `desktop/shared/html-validate.ts` checks unclosed/mismatched tags, duplicate ids and deprecated tags with line numbers (void elements, raw-text elements and implicit closes handled to avoid false positives); clicking an issue jumps to the CodeMirror source line.
- Network request waterfall: per-resource start/duration bars colored by type (MutationObserver + load/error capture timing, PerformanceResourceTiming preferred when available), as a list/waterfall view toggle inside the resource panel.
- Dark mode simulation: rewrites prefers-color-scheme media queries in inline styles and same-origin linked stylesheets, patches matchMedia and sets color-scheme, with 跟随系统 / 强制浅色 / 强制深色 toolbar modes.
- JS disable switch: reloads the preview with a `script-src 'none'` CSP (inspector bridge not injected) and reveals noscript content.
- In-preview text find (Ctrl+F): TreeWalker-based `<mark>` highlighting with a counter and previous/next navigation inside the iframe; CodeMirror keeps its own search.

## Markdown

- Mermaid render error panel: failed diagrams show a structured inline error block with the failing line number (parsed from the Mermaid parse error), details and the offending source line — Vditor fork untouched.
- Dead-link and missing-image scanner (检查死链): `desktop/main/markdown-link-scanner.ts` parses inline relative links/images (fence-aware, skips external/anchor/data targets), checks existence against the document directory and workspace root, and reports missing targets in a host panel; clicking reveals the line in the editor.
- Template system (从模板插入): built-in templates (blank / meeting notes / weekly report / README) plus user templates from `userData/markdown-templates/`; inserting replaces the content and marks the document dirty.
- Fix: inserting generated content (templates, AI TOC/summary) now re-enables the toolbar save button.

## AI assistant

- Edit and resend history messages: inline editing on user bubbles (Ctrl+Enter to submit), truncates later turns and re-attaches the original selection snapshot (`AssistantMessage.selectionSnapshot`).
- Ollama model discovery: the model field offers a datalist from the probed /api/tags model list, falling back to free input when unreachable.
- Model parameters: global temperature / maxTokens settings injected per protocol (OpenAI `temperature`/`max_tokens`, Anthropic, Gemini `generationConfig`, Ollama `options.num_predict`); CLI providers explicitly ignored.
- Pre-send sensitive data detection: `desktop/shared/sensitive-data.ts` rules (private key blocks, bearer/API keys, AWS keys, secret assignments, e-mails, checksum-validated Chinese ID numbers) trigger a masked confirmation bar before sending; cancelling keeps the draft.
- Settings dialog restructured into four tabs: 常规 / Provider / 动作 / 关于 (persistence schema unchanged).

## Fixes

- Main window background throttling disabled (`desktop/main/index.ts`): exports reveal their output in File Explorer, which can fully cover the app window; Windows occlusion tracking then froze rAF/IntersectionObserver and the app's editors/animations appeared dead while still alive.
