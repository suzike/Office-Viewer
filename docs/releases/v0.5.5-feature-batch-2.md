# Office Viewer Desktop 0.5.5 — Feature Batch 2

Release date: 2026-07-23

Second feature batch from the three module roadmaps (`docs/plans/`): 16 new features across HTML, Markdown and the AI assistant.

## HTML

- Console panel: captures console log/warn/error plus uncaught exceptions from the sandboxed preview through an injected inspector bridge (`desktop/main/html-inspector-script.ts`), level-colored and clearable (500-entry cap).
- Resource list panel: name/type/size/duration/status of page resources. PerformanceResourceTiming is unavailable for sandboxed opaque origins, so the inspector falls back to a DOM scan plus probe fetches.
- Performance metrics bar: DCL / FCP / LCP / resource count and total bytes via PerformanceObserver (`Timing-Allow-Origin` added to the protocol response).
- Responsive device presets: Desktop / iPhone 390×844 / iPad 820×1180 / custom size with a device frame, coexisting with the 50–200% preview zoom.
- Full-page screenshot PNG export (导出 PNG) through the hidden-window capturePage pipeline, written next to the source file (`exportHtmlImage` IPC).

## Markdown

- Export plain text (导出纯文本): `desktop/main/markdown-plain-text.ts` strips Markdown syntax (front-matter optional) and writes a .txt next to the source file.
- Focus mode (专注模式): typewriter scrolling with the current block highlighted, pure iframe CSS/JS — the Vditor fork is untouched.
- Zen mode (禅模式): distraction-free fullscreen, ESC to exit.
- Drag any file into the editor to insert a relative `[name](path)` link (main-process path resolution via `webUtils.getPathForFile`); images still go through the image service.
- AI one-click TOC / summary generation (AI 生成目录 / AI 生成摘要) inserted at the top of the document through the streaming polish pipeline.

## AI assistant

- Selection floating action bar (解释 / 翻译 / 引用到助手) over the text, Markdown and HTML viewers, wired through a cross-iframe selection bus (`selectionEvents.ts`); other viewers can opt in later.
- Custom quick actions: create, edit and reorder in the settings dialog (persisted, cap 50), merged into the quick-action bar and the slash-command palette.
- Prompt library: save frequent prompts (cap 100) and insert them into the composer from a panel.
- Global invoke shortcut (Ctrl+Shift+Space, toggleable) focuses the main window and opens the assistant panel; unregistered on quit.
- Real network probing for HTTP providers: latency and model lists (`/v1/models`, Ollama `/api/tags`) shown in the settings dialog; non-forced probes stay lazy/offline.
- Custom system prompt profile: persona / output language / style injected ahead of the safety rules.

## Fixes

- Word editor: the antd fullscreen Spin mask stayed mounted while fading out and intercepted hit-testing, swallowing the first click after loading. The mask is now unmounted as soon as loading ends (`src/react/view/word/Word.tsx`).

## Validation

- Desktop TypeScript typecheck passed; extension, renderer and host builds passed.
- All 75 desktop unit tests passed (4 new for the HTML inspector script, plain-text export and drop links; 4 new for assistant probing/persona/settings).
- Full Electron e2e suite passed: 16/16, including HTML viewer (batch 2 panels, device presets, PNG magic bytes), Markdown viewer (focus/zen, plain-text export, AI error path), AI assistant (custom actions, prompt library, selection bar, persona injection) and the Word/PowerPoint roundtrip gate.
- Batch 1 smoke suites re-run without regressions (`scripts/smoke-html-batch1.mjs`, `scripts/smoke-markdown-batch1.mjs`).

## Signing

The executables are not Authenticode-signed. Windows may display a SmartScreen warning until a trusted code-signing certificate is configured.
