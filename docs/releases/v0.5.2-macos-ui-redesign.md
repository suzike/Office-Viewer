# Office Viewer Desktop 0.5.2 — macOS Design Language Redesign

Release date: 2026-07-23

This release completely redesigns the desktop application with an Apple macOS design language covering visuals, color and interaction.

## Changes

- Replace the brutalist "paper ledger" theme with the macOS system palette (light `#f5f5f7` / dark `#1e1e20`), macOS blue accent (`#007aff` light / `#0a84ff` dark) and macOS semantic colors.
- Adopt the SF-style system font stack, a unified corner-radius scale (6/8/12px), soft ambient shadows and frosted `backdrop-filter` translucency for the header, menus, status bar and assistant panel.
- Style the frameless window controls as macOS traffic lights (previously unstyled), with hover glyphs and pressed feedback.
- Restyle menus as frosted rounded popovers, document tabs as Safari-style floating capsules, and file-type tokens as tinted rounded badges.
- Rework the welcome screen, metadata bar, status bar, state panels and drop overlay with Finder/inspector-inspired typography, macOS focus rings and smooth transitions.
- Rebuild the AI assistant launcher, panel, chat bubbles, composer and settings dialog in a macOS Notification Center style.
- Add a desktop-only Ant Design theme (`getDesktopAntThemeConfig`) so dialogs and inputs match the macOS shell; the VS Code extension theme is unchanged.
- Re-map `--vscode-*` viewer theme variables so embedded document viewers follow the desktop light/dark macOS theme without touching viewer code.
- Add `scripts/screenshot-desktop-ui.mjs` for headless UI screenshot verification.
- Fix Claude Code assistant requests failing (`--verbose` is required for `--print --output-format stream-json`) and retry without version-specific flags on older CLIs.
- Surface Codex stdout failures (`turn.failed`, non-transient `error` events) in the assistant panel instead of a generic exit-code message, and deduplicate Claude streamed deltas vs final snapshots.
- Pass proxy / API-key / certificate environment variables to local CLI providers and discover executables in more install layouts (`OpenAI\Codex\bin`, `%USERPROFILE%\.local\bin`).
- Show the probe failure reason for unavailable local providers in the assistant model row.
- Render assistant replies as rich Markdown (headings, lists, tables, syntax-highlighted code blocks, quotes) via markdown-it with escaped raw HTML, styled to match the macOS assistant panel.
