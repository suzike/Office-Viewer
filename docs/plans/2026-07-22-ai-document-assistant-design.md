# AI 文档交互助手设计与决策

状态：已实施首个完整纵向版本（2026-07-22）

## 目标

在独立 Windows Office Viewer 中增加右下角文档助手，统一使用本机 Codex CLI、Claude Code CLI、Ollama，以及 OpenAI-compatible、Anthropic、Gemini 等第三方接口。助手默认只做文档理解与内容生成，不直接修改文件。

## ADR-001：主进程统一 AI 网关

- 决策：渲染进程只提交提供器 ID、当前文档会话 ID 和对话；文档提取、密钥解密、网络请求、CLI 子进程都在 Electron 主进程中完成。
- 原因：避免 API Key 和本地文件路径处理能力暴露给页面；统一超时、取消、输出上限、SSRF 防护和错误脱敏。
- 放弃方案：页面直接请求第三方 API。该方案实现简单，但会把密钥交给渲染层，无法接受。
- 放弃方案：每个文档查看器各自实现 AI。该方案会复制提取逻辑，并产生不一致的安全边界。

## ADR-002：本地 CLI 使用临时无状态会话

- Claude Code：启用安全模式、禁用工具、禁用会话持久化、禁止交互式权限请求。
- Codex CLI：隔离工作目录、只读沙箱、禁审批、临时会话、忽略项目规则和用户扩展配置。
- 关键限制：当前 Codex CLI 没有公开的“禁用所有读取工具”参数，因此 UI 将其标为高信任实验提供器；不能把它描述成纯聊天沙箱。
- 文档正文只通过 stdin 传递，不进入命令行参数、临时文件或应用日志。

## ADR-003：提供器与密钥

- 内置：Codex、Claude Code、DeepSeek、Kimi、Ollama。
- 可定义：OpenAI-compatible、Anthropic、Gemini、Ollama 兼容端点。
- DeepSeek 使用其官方 OpenAI-compatible 基础地址 `https://api.deepseek.com`，默认模型 `deepseek-v4-flash`。
- Kimi 使用其官方 OpenAI-compatible 基础地址 `https://api.moonshot.cn/v1`，默认模型 `kimi-k2.6`。
- API Key 使用 Electron `safeStorage` 交给 Windows 加密；JSON 设置只存 `hasApiKey` 状态，渲染进程永远拿不到密钥值。

## 文档上下文策略

| 格式 | 当前策略 |
| --- | --- |
| Markdown、HTML、HTTP、CSV、TSV、代码与常见文本 | 直接解码文本 |
| DOCX/DOTX | 提取正文段落及脚注、尾注、批注文本 |
| PPTX/PPTM | 按幻灯片顺序提取文本运行 |
| XLS/XLSX/XLSM/ODS | 按工作表转换为结构化 CSV 文本 |
| PDF | 使用 PDF.js 提取逐页文本层；扫描版明确提示需要 OCR |
| 图像、压缩包、字体、PSD、Parquet、XMind 等 | 当前只提供元数据，不声称已理解二进制正文 |

上下文默认上限 160,000 字符，可在 8,000 至 500,000 之间配置。超长文档保留头部和尾部，并明确标记中间截断。

## 安全与故障边界

- 文档内容被包在 `DOCUMENT_DATA` 中并标记为不可信数据，禁止把文档内提示当成系统指令。
- HTTP 提供器禁止凭据出现在 URL，禁止重定向，始终阻断元数据和链路本地地址。
- 本机或局域网地址必须由用户逐提供器显式允许。
- 单条消息 64 KB、会话 400 KB、模型输出 2 MB、提供器原始响应 8 MB、默认超时 180 秒。
- CLI 使用原生 `.exe`、`shell:false`、隔离目录和环境变量白名单；取消时终止已追踪进程树。
- 第三方接口“检测”只做本地配置完整性检查，不擅自发送计费请求。

## 交互能力

- 流式多轮问答、停止、重试、复制、清空会话。
- 总结、解释选区、提取大纲、翻译、文档审查、表格洞察。
- 提取关键事实、生成行动项、风险登记、评审问题清单。
- 模型选择与状态、提供器增删改、上下文上限、局域网权限设置。
- `Ctrl+Shift+I` 开关助手，`Ctrl+Enter` 发送。

## 后续增强清单

- OCR：为扫描 PDF 和图像增加本地 OCR，需单独评估模型体积与语言包。
- 文档选区桥接：不同查看器采用 iframe/Canvas/编辑器，需为每类查看器定义统一选区事件。
- 引用定位：让模型返回结构化引用，并映射到 PDF 页、PPT 页、工作表单元格或 Markdown 标题。
- 写回工作流：仅在可预览 diff 且用户明确确认后写入，不开放模型直接文件权限。
- 多文档对比与本地索引：需要独立的范围选择、索引清除和隐私说明。
